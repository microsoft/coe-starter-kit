## How to Use This Documentation

### If You're Having an Issue

1. Start with [TROUBLESHOOTING.md](TROUBLESHOOTING.md)
2. Find your issue in the Table of Contents
3. Follow the solutions in order
4. If you still need help, create an issue with details of what you've tried

### If You're Responding to an Issue

1. Check [COMMON-ISSUE-RESPONSES.md](COMMON-ISSUE-RESPONSES.md) for a template
2. Copy the relevant template
3. Customize it for the specific user's situation
4. Link to [TROUBLESHOOTING.md](TROUBLESHOOTING.md) for detailed steps

### If You're Contributing

1. Review [SOLUTION-SUMMARY.md](SOLUTION-SUMMARY.md) to understand the approach
2. Add new sections to [TROUBLESHOOTING.md](TROUBLESHOOTING.md) as issues are identified
3. Create new templates in [COMMON-ISSUE-RESPONSES.md](COMMON-ISSUE-RESPONSES.md) for recurring issues
4. Keep the documentation user-friendly and actionable

## Documentation Standards

When adding to these documents:
- Use clear, simple language
- Provide step-by-step instructions
- Include screenshots when helpful
- Link to official Microsoft documentation
- Clarify support boundaries (what's supported vs. best-effort)
- Test all solutions before documenting them
