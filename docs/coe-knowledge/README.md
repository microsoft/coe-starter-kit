# CoE Starter Kit Knowledge Base

This directory contains documentation resources for the CoE Starter Kit community, maintainers, and contributors.

## Purpose

These documents serve multiple purposes:

1. **Troubleshooting Guides**: Step-by-step instructions for resolving common issues
2. **GitHub Response Templates**: Pre-written responses for common questions and issues raised in GitHub
3. **Best Practices**: Recommended patterns and practices for deploying and maintaining the CoE Starter Kit
4. **Known Issues and Limitations**: Documented limitations and workarounds

## Documents

### [COE-Kit-Common-GitHub-Responses.md](./COE-Kit-Common-GitHub-Responses.md)

A playbook of ready-to-use explanations, limits, and workarounds for common issues reported in GitHub issues. This document is intended to help maintainers and community members provide consistent, accurate responses to frequently asked questions.

**Topics Covered:**
- Dashboard and inventory issues
- Setup and configuration problems
- Known limitations and support model
- Licensing and pagination requirements
- Data Lake (BYODL) guidance
- Language and localization requirements
- Cleanup and maintenance procedures

**How to Use:**
- Search for the issue category that matches the GitHub issue
- Copy the relevant section as a starting point for your response
- Customize with specific details from the issue
- Include links to official documentation where appropriate

### [Troubleshooting-Inventory-and-Dashboards.md](./Troubleshooting-Inventory-and-Dashboards.md)

A comprehensive troubleshooting guide focused on the most common issues users face: inventory collection not working and dashboards not showing data.

**Topics Covered:**
- Quick diagnostic checklist
- Step-by-step first-time setup instructions
- Detailed troubleshooting for inventory collection failures
- Power BI connection and refresh issues
- Partial or incomplete data problems
- Understanding inventory flow architecture
- Monitoring and validation procedures

**How to Use:**
- Share the link with users experiencing inventory/dashboard issues
- Walk through the quick diagnostic checklist to identify the problem area
- Follow the detailed steps for the specific issue category
- Use the architecture section to understand how inventory works
- Reference when creating GitHub issue responses

## Contributing

If you have suggestions for improving these documents or want to add new troubleshooting guides:

1. Review the [HOW_TO_CONTRIBUTE.md](../../HOW_TO_CONTRIBUTE.md) guide in the repository root
2. Create a branch with your changes
3. Submit a pull request with a clear description of your additions/changes
4. Reference any related GitHub issues that your documentation addresses

### Guidelines for Documentation

When contributing to this knowledge base:

- **Be specific**: Include exact flow names, table names, and error messages
- **Include steps**: Provide numbered, actionable steps
- **Add context**: Explain why something happens, not just how to fix it
- **Link to official docs**: Reference Microsoft Learn documentation where appropriate
- **Update dates**: Add "Last Updated" dates to documents you modify
- **Test instructions**: Verify that troubleshooting steps actually work before documenting them

## Maintenance

These documents should be reviewed and updated:

- **After major CoE Starter Kit releases**: Verify steps are still accurate
- **When new common issues emerge**: Add new troubleshooting sections
- **When official documentation changes**: Update links and references
- **When community feedback suggests improvements**: Incorporate lessons learned
